## Category 20 — Materiality Filter for CFO/Board (surface only material — not everything)

| Severity | Finding | Evidence | Best-in-class gap | Recommendation |
|----------|---------|----------|-------------------|----------------|
| **Blocker** | No tiered materiality model exists. The flux threshold (percent + dollar, OR/AND) is a single-level ops-accounting filter. The AI draft service applies no additional board-level filter; it simply takes top-8 by `Math.Abs(VarianceAmount)` regardless of whether an item is board-relevant. | XeroLedgerServices.cs:2060–2065: `.OrderByDescending(x => Math.Abs(x.VarianceAmount)).Take(8)`. | **Major** | The AI draft suggestion kind is always `"FluxContext"` (XeroLedgerServices.cs:2075) — a single flat category. There is no distinction between executive-summary-level items ("board" severity) and operational footnote items ("ops" severity). ## Pillar Verdict — AI Board Package

The schema foundation (Package → Slide → Block, versioned snapshots, draft suggestion workflow) is sound. However the marquee differentiator — AI proposes slide-level keep/modify/add/remove against last month's baseline — **does not exist in any form**. `BaseFrom` is a display label, not a functional link to the prior package. The draft service generates add-only suggestions from current-month flux with no cross-month awareness. There is no tiered materiality separating board items from ops detail. Every AI-generated block risks wholesale rewriting prior commentary with no deduplication guard.

**Blockers: 3 | Majors: 4**

---

## Top 3 Fixes

1. **Implement the cross-package diff engine** (Blocker — Category 19): Add `PriorPackageId` FK to `ReportPackage`, build `PackageDiffService` that reads both packages' slide snapshots and emits typed `SlideDecision` (keep/modify/add/remove) records. Wire `AiPackageDraftService.CreateDraftsAsync` to consume this diff before generating suggestions. This is the foundational requirement for the marquee feature.

2. **Introduce tiered board-level materiality** (Blocker — Category 20): Add `BoardDollarThreshold` / `BoardPercentThreshold` to `FluxReviewGroup` or a `MaterialityPolicy` entity. Filter AI draft suggestions through the board threshold before they reach the slide editor, relegating below-threshold items to an appendix. Assign differentiated `Kind` values (`BoardMaterial`, `OpsDetail`, `FYI`) so the UI can render them at the correct hierarchy level.

3. **Add unchanged-narrative carry-forward and deduplication guard** (Blocker — Category 19 / Major — Category 20): For slides where month-over-month variance is below the board threshold, emit explicit `keep` draft records (not silence) so the CFO consciously approves prior commentary. Before appending any AI-generated block, hash-compare against existing slide blocks and skip if below a similarity delta threshold, preventing wholesale narrative rewrites.
